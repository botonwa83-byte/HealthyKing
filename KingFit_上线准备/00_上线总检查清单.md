# KingFit 上线总检查清单（App Store Connect）

> 版本 1.0（build 1） · Bundle ID `com.healthyking.app` · 最低 iOS 17.0 / watchOS 10.0 · iPhone + Apple Watch
> 本清单配套文件均在本文件夹内。打 ✅ 表示已就绪，⬜ 表示需在 ASC 后台、代码仓库或真机环境完成。

## A. 阻断级（不处理容易被拒）

- ⬜ **隐私政策 URL 未上线**
  - 健康类 App 必须提供清晰、可访问的隐私政策。
  - 建议创建 `docs/privacy.html`、`docs/terms.html`、`docs/support.html` 并用 GitHub Pages 托管。
  - 正文草案见 `02_隐私政策_PrivacyPolicy.md`，托管步骤见 `09_GitHub_Pages_托管说明.md`。

- ✅ **`PrivacyInfo.xcprivacy` 已创建**
  - 当前 App 使用 `@AppStorage("hasCompletedOnboarding")`，底层属于 UserDefaults。
  - 已新增 `Sources/SharedPrivacy/PrivacyInfo.xcprivacy`，声明 `NSPrivacyAccessedAPICategoryUserDefaults`，理由码 `CA92.1`，`NSPrivacyTracking = false`，`NSPrivacyCollectedDataTypes = []`。
  - 如果后续加入网络日志、分析 SDK、崩溃 SDK 或云同步，需要同步扩展隐私清单。

- ✅ **HealthKit 权限说明与审核备注需保持一致**
  - 读取：HRV、静息心率、呼吸率、血氧、VO2 Max、体重、心率、步数、活动能量、Apple 锻炼时间、睡眠、出生日期、生理性别、锻炼记录。
  - 写入：无。当前版本没有体重录入界面，已移除体重写入权限请求。
  - 目的：本机生成健康趋势、恢复评分、训练负荷、睡眠详情和活动概览。
  - 不能写成医疗诊断、疾病风险评估、治疗建议。

- ⬜ **`DEVELOPMENT_TEAM` 核实**
  - 当前 `project.yml` 里 `DEVELOPMENT_TEAM: ""`。
  - Archive 上传前请在 Xcode → Signing & Capabilities 选择你的开发者团队，并确认 iOS、watchOS、Widget 三个 target 都能签名。

- ✅ **开发者其他应用入口已处理**
  - 已移除旧的占位推广卡片，不再保留 `https://apps.apple.com/` 这种空链接。
  - 设置页底部保留“发现我的其他应用”，打开固定 App Store 开发者主页。
  - 该入口不读取 HealthKit 数据，不做个性化推荐，不在健康评分或训练负荷结果中插入推广。

- ✅ **训练负荷页“风险”措辞已降级**
  - 已将 `Sources/HealthyKing/Views/TrainingLoad/TrainingLoadView.swift` 中的“较高风险参考线/高风险阈值”改为“较高负荷参考线”。

## B. 提交前应处理

- ⬜ **真机 HealthKit 测试**
  - 首次启动能正常弹出 HealthKit 授权。
  - 拒绝权限后 App 有清晰错误状态，不崩溃。
  - 只授权部分项目时，缺失指标显示“暂无数据/数据积累中”，不误导。
  - 当前版本不请求写入权限；体重仅作为读取指标展示。

- ⬜ **Apple Watch 测试**
  - watchOS App 可独立请求 HealthKit 授权并加载数据。
  - 表盘组件/Widget 不主动触发权限弹窗，只读取已有授权下的数据。
  - 无数据时组件显示占位，不崩溃。

- ⬜ **健康免责声明检查**
  - Onboarding 已有免责声明：仅供一般健康与运动趋势参考，不用于诊断、治疗、治愈或预防疾病。
  - 建议截图、商品描述、审核备注都保持同一口径。

- ⬜ **截图补齐**
  - iPhone 6.9" 截图必需。
  - Apple Watch 截图建议上传。
  - 建议取景见 `08_截图与素材清单.md`。
